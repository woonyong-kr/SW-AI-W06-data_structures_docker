# Week 6 핵심 역량 목표

기간: 2026-04-03 ~ 2026-04-09

이번 주는 팀원 전체가 포인터와 동적 할당을 이해하고,
기본 자료구조를 직접 구현한 뒤 왜 그렇게 구현했는지 설명할 수 있는 상태를 목표로 잡는다.

나중에 계산하기 편하게 항목은 숫자로 적어둔다.

## 1. 문제해결

- 목표: Linked List 7문제 + SQL 처리기 최소 기능 2개(`INSERT`, `SELECT`) = 9개
- 계산: (완료 수 / 9) x 100

## 2. 설계

- 목표: SQL 처리기 설계 포인트 5개 정리
- 기준: 입력 흐름, 파싱 구조, 내부 데이터 표현, 실행 흐름, 파일 저장 포맷
- 계산: (정리한 포인트 수 / 5) x 100

## 3. 구현

- 목표: Linked List 7문제 구현 + SQL 처리기 핵심 기능 5개 구현 = 12개
- 기준: 파일 입력, `INSERT` 파싱, `SELECT` 파싱, `INSERT` 실행, `SELECT` 실행
- 계산: (완료 수 / 12) x 100

## 4. 품질

- 목표: Linked List 7문제 자체 검증 + SQL 처리기 테스트 5개 이상 = 12개
- 기준: 정상 입력, 빈 입력, 잘못된 SQL, 다중 행 처리, 저장/조회 일관성
- 계산: (검증 완료 수 / 12) x 100

## 5. 유지보수

- 목표: 5일 이상 의미 있는 커밋 + 문서/README 업데이트 2회 이상 = 7개
- 계산: (완료 수 / 7) x 100

## 6. 협업

- 목표: 협업 룰 합의 1건 + 정기 코어타임 5회 이상 + 수요 코딩회 역할 분담/통합 1회 = 7개
- 계산: (완료 수 / 7) x 100

## 7. 태도

- 목표: 공통 문서 5개 작성 + 코어타임 설명/공유 3회 이상 = 8개
- 기준: 포인터, 메모리 구조, 동적 할당, SQL 처리기 로직 중 1개 이상 직접 설명
- 계산: (완료 수 / 8) x 100

## 8. 비즈니스 이해

- 목표: 파일 기반 SQL 처리기가 왜 필요한지, 실제 DBMS와 차이가 뭔지, 이번 구현 한계가 뭔지 3개 이상 정리
- 계산: (정리한 포인트 수 / 3) x 100

## 9. AI 활용

- 목표: AI 활용 원칙 정리 1건 + AI 도움받은 핵심 로직 3건 이상 설명/검증 = 4개
- 계산: (완료 수 / 4) x 100

## 10. 학습 민첩성

- 목표: 아래 7개 주제를 설명 가능한 수준까지 보기
- 주제: C 기본 문법, 포인터, 구조체, `malloc/free`, 메모리 구조, `gcc/gdb/valgrind`, 파일 I/O
- 계산: (설명 가능한 주제 수 / 7) x 100

## 메모

- C문법은 익숙하지 않은 사람만 보고, 총 8시간 넘기지 않는다.
- 문법 전체를 다 보는 게 목표가 아니라 과제 진행 가능한 수준이면 된다.
- Data Structures는 Linked List까지는 전부 구현하는 걸 최소 기준으로 잡는다.
- 수요 코딩회는 구현 우선이지만, 핵심 로직은 반드시 직접 설명할 수 있어야 한다.
