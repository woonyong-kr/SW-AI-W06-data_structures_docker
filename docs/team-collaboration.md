## 6. 코어타임 운영 포인트

- 포인터와 동적 할당은 말로 설명하고, 필요하면 그림과 예제로 다시 정리합니다.
- Linked List 구현은 단순 정답 공유보다 노드 연결과 해제 흐름을 중심으로 리뷰합니다.
- SQL 처리기 구현 전에는 입력 -> 파싱 -> 실행 -> 저장 흐름을 먼저 말로 맞춰 봅니다.
- 테스트 케이스와 엣지 케이스는 구현 후가 아니라 구현 중간부터 함께 정리합니다.

## 8. 협업 팁

- 설명이 안 되는 코드는 아직 내 코드가 아니라는 기준으로 다시 확인합니다.
- 포인터 문제는 값보다 "누가 무엇을 가리키는지"를 그림으로 공유하면 훨씬 빠르게 정리됩니다.
- 메모리 해제는 "어디서 할 수 있는가"보다 "누가 해제 책임을 가지는가"를 먼저 정합니다.
- 발표는 README 기준으로 진행되므로, 구현 과정에서 설계 포인트와 테스트 결과를 꾸준히 문서화합니다.
