# Week 6 - 핵심 역량 목표 설정

> 작성 시점: 주차 시작 시
>
> 적용 기간: 2026-04-03 ~ 2026-04-09

이번 주 핵심 목표는 팀원 전체가 C의 포인터와 동적 할당을 이해하고, 기본 자료구조를 직접 구현하며, 구현 이유를 스스로 설명할 수 있는 상태에 도달하는 것입니다.

달성률 산정을 위해 목표는 가능한 한 수치화된 형태로 작성합니다.

## 1. 문제해결

- 목표: Linked List 7문제 전체 해결 + SQL 처리기 최소 기능 2개(`INSERT`, `SELECT`) 해결 = 총 9개 항목 완료
- 달성률 산식: (완료한 항목 수 / 9) × 100

## 2. 설계

- 목표: SQL 처리기의 핵심 설계 포인트 5개를 README 또는 설계 메모에 명시
- 설계 포인트 기준: CLI 입력 흐름, SQL 파싱 구조, 내부 데이터 표현, 실행 흐름, 파일 저장 포맷
- 달성률 산식: (문서에 명시한 설계 포인트 수 / 5) × 100

## 3. 구현

- 목표: Linked List 7문제 구현 + SQL 처리기 핵심 기능 5개 구현 = 총 12개 구현 항목 완료
- 구현 항목 기준: 파일 입력, `INSERT` 파싱, `SELECT` 파싱, `INSERT` 실행, `SELECT` 실행
- 달성률 산식: (완료한 구현 항목 수 / 12) × 100

## 4. 품질

- 목표: Linked List 7문제에 대한 자체 검증 완료 + SQL 처리기 테스트 5개 이상 작성 및 통과 = 총 12개 항목 완료
- 테스트 기준: 정상 입력, 빈 입력, 잘못된 SQL, 다중 행 처리, 파일 저장/조회 일관성
- 달성률 산식: (검증 완료 항목 수 / 12) × 100

## 5. 유지보수

- 목표: 이번 주 5일 이상 의미 있는 커밋 수행 + README/문서 업데이트 2회 이상 = 총 7개 항목 완료
- 달성률 산식: (완료한 항목 수 / 7) × 100

## 6. 협업

- 목표: 팀 협업 룰 합의 1건 + 정기 코어타임 5회 이상 참여 + 수요 코딩회 역할 분담/통합 1회 완료 = 총 7개 항목 완료
- 달성률 산식: (완료한 항목 수 / 7) × 100

## 7. 태도

- 목표: 공통 문서 5개 작성 완료 + 코어타임에서 설명/공유 3회 이상 수행 = 총 8개 항목 완료
- 설명/공유 기준: 포인터, 메모리 구조, 동적 할당, SQL 처리기 핵심 로직 중 1개 이상을 직접 발표
- 달성률 산식: (완료한 항목 수 / 8) × 100

## 8. 비즈니스 이해

- 목표: 파일 기반 SQL 처리기가 왜 필요한지, 실제 DBMS와 어떤 차이가 있는지, 이번 구현의 한계가 무엇인지 3개 포인트 이상 README에 명시
- 달성률 산식: (명시한 포인트 수 / 3) × 100

## 9. AI 활용

- 목표: AI 활용 원칙 수립 1건 + AI 도움을 받은 핵심 로직 3건 이상에 대해 원리 설명 및 검증 완료 = 총 4개 항목 완료
- 달성률 산식: (완료한 항목 수 / 4) × 100

## 10. 학습 민첩성

- 목표: 아래 7개 주제 중 7개 모두에 대해 학습 후 설명 가능한 수준 도달
- 주제 기준: C 기본 문법, 포인터, 구조체, `malloc/free`, 메모리 구조, `gcc/gdb/valgrind`, 파일 I/O
- 달성률 산식: (설명 가능한 주제 수 / 7) × 100

## 공통 메모

- C언어 문법 학습은 익숙하지 않은 사람만 진행하고, 총 학습 시간은 8시간 이내로 제한합니다.
- C언어 문법 전체를 마스터하는 것이 목표가 아니라, 과제와 구현 과정에서 필요한 부분을 익히는 것을 목표로 합니다.
- Data Structures는 Linked List까지 모두 구현하는 것을 최소 기준으로 삼습니다.
- 수요 코딩회에서는 학습보다 구현을 우선하되, 핵심 로직은 반드시 팀원이 직접 설명할 수 있어야 합니다.
